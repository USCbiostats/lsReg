## Package Overview

`lsReg` is an R package for large-scale regression. It efficiently fits models where a small number of covariates (`xr`) are repeatedly tested against a fixed base model (`xl`), without re-fitting the full model each time. The canonical use case is GWAS (genome-wide association studies).

Supported families: `gaussian`, `binomial`, `poisson`.  
Supported test types: `lrt`, `score`, `robustscore`, `wald`, `robustwald`.

## Architecture

The package has a two-phase API:

1. **Allocation phase** — `lsReg(basemdl, colstoadd, testtype)` pre-computes and caches expensive quantities from the base GLM (QR decomposition, residuals, info matrices). Returns an `lsregmem` S3 object.

2. **Test phase** — `runtest(lsregmem, xr)` uses the cached memory to quickly compute the test statistic for a new column matrix `xr`.

### Call stack

```
lsReg()                        # R/lsReg.R — validates inputs, dispatches by family
  └─ allocate.lsregmem()       # R/memory.R — routes to family-specific allocator
       ├─ allocate.lslinregmem()   # R/linregmem.R
       ├─ allocate.lslogregmem()   # R/logregmem.R
       └─ allocate.lspoisregmem()  # R/poisregmem.R
           └─ allocate.ls*regmem.<testtype>()  # one per test type per family
                └─ init*() C++ function via Rcpp  # precomputes base-model quantities

runtest(lsregmem, xr)          # R/teststat.R — dispatches by family + testtype
  ├─ ls*regfit() C++ — fits full model using cached QR
  └─ ls*reg<testtype>() C++ — computes test statistic
```

### C++ / Rcpp layer (`src/`)

Each source file covers one regression family:
- `linear.cpp` — all linear regression routines
- `logistic.cpp` — logistic regression routines
- `poisson.cpp` — Poisson regression routines

`RcppExports.cpp` and `R/RcppExports.R` are auto-generated by `Rcpp::compileAttributes()` — do not edit manually.

### Memory objects (`lsregmem` class)

Each allocator builds a named list of pre-allocated matrices/vectors. The C++ routines mutate these in-place (pass-by-reference via Rcpp), avoiding repeated memory allocation across thousands of `runtest()` calls. The `fitdata` sub-list holds QR decomposition results for the base model.

### Test data

Sample fitted models and datasets are in `inst/extdata/` as `.rds` files:
- `linmodel.rds` — a pre-fitted Gaussian GLM used in tests and examples
- `subdata.rds`, `subdata2.rds` — subject data matrices
